## 规范

我做事通常不喜欢遵循固定的规范，尤其这个博客仓库我将会耕耘很长时间。但是规范意味着可以进行一些自动化处理，比如可能会做一些中英文混排优化、图片转base64、引用格式优化等等。

下面这些简洁有力的规范完全符合我的价值观。

### 内容格式: common markdown

在 markdown 世界中，`强大的语法功能` 和 `可迁移性` 不可得兼，对于我的情况，我总是会尝试不同的编辑器，所以为了保证可迁移性，我的博客所有内容都必须为common markdown。

如果你对 markdown 的扩展语法和变体感兴趣（其实是我对知识面的卖弄），可以试试了解这些名词: block-style (editor.js), Obsidian, WYSIWYG (milkdown, typora), latex 和 mermaid 。


### 分组方式: 时间

如何呈现博客内所有文章的简概？一般博客使用tag和分类，进一步还有子级分类和子tag，obsidian社区喜欢用网状组织，还有些博客喜欢用词云之类的方式表达内容的“分布情况”，类似描述性统计一样。

和第一条规范类似的逻辑，我不遵守固定的呈现和表达方式，但我可以提供一个分组方式，组内按当时的喜好处理。这条规范带来的好处是可能会方便我为每个组添加一些可视化的内容。
